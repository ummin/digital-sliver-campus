# 디지털 실버 캠퍼스 MVP PRD

## 🎯 핵심 정보

**목적**: 디지털 기기에 익숙하지 않은 중장년·노년층이 키오스크·스마트폰 앱·AI 도구 사용법을 가장 쉬운 단계부터 따라 하며 익히도록 돕는다.
**사용자**: 50대 이상 중장년/정년퇴임 세대 학습자 (디지털 기기 사용에 서툰 비로그인 방문자)

## 🚶 사용자 여정

```
1. 홈
   ↓ "배우고 싶은 주제 선택" 클릭

2. 카테고리 목록 (키오스크 / 스마트폰 앱 / AI 도구)
   ↓ 카테고리 선택

3. 실습 콘텐츠 목록 (쉬운 난이도 순으로 정렬)
   ↓ 콘텐츠 카드 선택

4. 실습 콘텐츠 뷰어 (단계별 가이드)
   ↓ [이전/다음] 버튼으로 단계 이동
   ↓ 완료 시

   [궁금한 점 있음] → 피드백/문의 폼 → 제출 → 완료 안내
   [다른 주제 학습]  → 카테고리 목록으로 복귀
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 카테고리 탐색 | Notion에서 가져온 학습 카테고리(키오스크/앱/AI)를 큰 카드로 노출 | 학습 진입점, 주제 선택의 핵심 | 홈, 카테고리 목록 페이지 |
| **F002** | 난이도순 콘텐츠 목록 | 선택한 카테고리의 실습 콘텐츠를 난이도/순서 필드 기준 오름차순 정렬 노출 | "가장 쉬운 단계부터" 학습 구조의 핵심 | 실습 콘텐츠 목록 페이지 |
| **F003** | 단계별 실습 뷰어 | 단계별 설명·이미지/스크린샷·영상을 한 단계씩 큰 화면으로 안내, 이전/다음 이동 | 실습 학습의 본질, 클릭 단계 최소화 | 실습 콘텐츠 뷰어 페이지 |

## ⚙️ 구현 메모 (기술 제약 및 대응)

> Notion 공식 문서 기반 기술 검증에서 도출한 제약과 대응 방안. 개발 착수 전 반드시 반영.

### 1. Notion 호스팅 파일 URL 만료 (콘텐츠 미디어)
- **제약**: Notion이 호스팅하는 파일 URL은 fetch 시점부터 **1시간만 유효한 임시 URL**이며, 캐시·정적 참조 시 깨짐.
- **대응**: 실습 콘텐츠 본문의 이미지/영상은 **Vercel Blob에 호스팅하고 Notion에는 external URL로 참조**한다(external URL은 만료되지 않음). 부득이 Notion 호스팅 파일을 쓰는 경우 콘텐츠 페이지의 `revalidate`를 **3600초 미만**으로 설정해 URL을 주기적으로 재발급한다.

### 2. 피드백 첨부 파일 처리 (F004)
- **제약**: Notion API 직접 업로드는 무료 워크스페이스 기준 **파일당 5 MiB** 제한.
- **대응**: 첨부 파일은 **Vercel Blob에 업로드 후 영구 URL만 Notion 피드백 DB에 저장**한다. 폼 단계에서 Zod로 파일 크기·확장자(이미지 위주)를 검증한다.

### 3. 익명 제출 분기 (F004)
- **대응**: "익명" 선택 시 이름은 "익명"으로 고정하고 연락처 입력을 선택적으로 처리하도록 Zod 스키마에 조건부 검증(분기)을 둔다.

### 4. 필수 패키지 (미설치 → 설치 필요)
- `@notionhq/client` (Notion API), `@vercel/blob` (파일 스토리지) — 현재 `package.json`에 없으므로 구현 시 추가.
